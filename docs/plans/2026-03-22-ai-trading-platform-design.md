# AI量化交易平台设计文档

## 概述

构建一个开源的AI量化交易平台，支持从个人投资者到专业开发者的全用户群体，覆盖A股/美股/港股/加密货币/期货期权全市场。采用分层微服务架构，第一期本地Docker部署，后期扩展到K8s云端部署。

## 目标用户

- 个人投资者：提供简单易用的量化交易工具
- 量化研究员：提供专业的策略研发和回测平台
- 开发者社区：提供可扩展的AI交易框架和SDK

## 整体架构

## 分层架构设计

### 1. 数据层

统一数据网关适配多市场API，支持不同类型数据的最优存储方式。

**数据库配置**：

```python
DATABASES = {
    'market_data': {
        'ENGINE': 'timescaledb',  # 时序数据
        'NAME': 'market_data',
        'HOST': 'localhost',
        'PORT': 5432,
    },
    'user_data': {
        'ENGINE': 'sqlite3',      # 用户数据（本地模式）
        'NAME': 'user_data.db',
    },
    'strategy_data': {
        'ENGINE': 'redis',        # 策略缓存
        'HOST': 'localhost',
        'PORT': 6379,
    }
}
```

**开源数据源**：

| 数据类型 | 数据源 | API |
|----------|--------|-----|
| A股行情 | AKShare / Tushare | 开源免费 |
| 美股行情 | Yahoo Finance | yfinance |
| 港股行情 | AKShare | 开源免费 |
| 加密货币 | Binance API | 官方API |
| 期货期权 | AKShare | 开源免费 |

**数据流**：

```
外部API → Data Gateway → Cache Layer → SKILL Engine
                ↓
        Time Series Store ← User Data Service
```

### 2. AI引擎层

六个专项Agent协同工作，通过消息队列实现事件驱动通信。

```
┌─────────────────────────────────────────────────────────────┐
│                     AI Engine Layer                         │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │  Data Agent  │  │Analysis Agent│  │Strategy Agent│       │
│  │  数据采集     │  │  行情分析    │  │  策略生成    │       │
│  │  清洗存储     │  │  趋势预测    │  │  优化建议    │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
│                                                              │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐       │
│  │Trading Agent │  │ Risk Agent   │  │  Backtest    │       │
│  │  交易执行     │  │  风险评估    │  │   Agent      │       │
│  │  订单管理     │  │  仓位控制    │  │  回测验证    │       │
│  └──────────────┘  └──────────────┘  └──────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

**Agent职责**：

- **Data Agent**：数据采集、清洗、存储
- **Analysis Agent**：行情分析、趋势预测、情感分析
- **Strategy Agent**：策略生成、参数优化、建议输出
- **Trading Agent**：交易执行、订单管理、成交确认
- **Risk Agent**：风险评估、仓位控制、止损止盈
- **Backtest Agent**：历史回测、策略验证、性能评估

**通信机制**：

- 消息队列：Redis Stream（轻量级，本地部署友好）
- 事件驱动：市场数据更新 → Data Agent → Analysis Agent → Strategy Agent
- 编排引擎：类似 finsrecopilot 的插件注册模式

```python
        """版本号"""
        pass

    def execute(self, context: SkillContext) -> SkillResult:
        """执行入口"""
        pass

    def validate(self, params: dict) -> bool:
        """参数校验"""
        pass
```

**Marketplace功能**：

- 用户上传/下载 SKILL
- 版本管理和依赖解析
- 评分和评论系统

## 技术栈选型

| 组件 | 技术选型 | 理由 |
|------|---------|------|
| 前端框架 | React + TypeScript | 生态成熟，类型安全 |
| UI组件库 | Ant Design / shadcn/ui | 开箱即用的交易风格组件 |
| 后端框架 | FastAPI (Python) | 异步高性能，AI生态友好 |
| 数据库 | SQLite + TimescaleDB | 本地轻量 + 时序数据支持 |
| 缓存 | Redis | 实时数据缓存 |
| 消息队列 | Redis Stream | 轻量级事件驱动 |

## 风险与约束

### 技术风险

1. **实时数据延迟**：开源数据源可能有延迟，影响交易决策
2. **AI模型准确性**：开源模型在金融场景的准确性有限
3. **回测过拟合**：策略回测可能存在过拟合问题

### 法律合规

1. **交易接口限制**：不同券商API接入需要合规审批
2. **数据使用合规**：需遵守各数据源的使用协议
3. **免责声明**：平台仅供学习研究，不构成投资建议

### 安全风险

1. **API Key 管理**：用户的交易API需要加密存储
2. **策略隐私**：用户的自定义策略需要保护
3. **资金安全**：交易执行需要风控机制

## 成功指标

1. **功能完整性**：支持 5 大市场的数据获取和回测
2. **性能指标**：回测速度 > 1000条/秒，实时行情延迟 < 1秒
3. **社区活跃度**：SKILL Marketplace 上线 3 个月内收录 50+ 精品 SKILL
4. **用户增长**：上线 6 个月内达到 1000+ GitHub Stars

## 参考设计

本项目参考了 finsrecopilot 的插件化中间件设计模式：

- 插件化配置：通过 AppConfig 进行模块注册
- 中间件模式：拦截处理请求
- 统一接口：标准化的接口定义
- 多环境支持：dev/test/prod 配置分离
